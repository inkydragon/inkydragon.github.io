# Amazon 叙事法核心原则（论文笔记版）

> 当你需要理解为什么这些栏目设计成这样时读这份。

## 三条元原则

Amazon 叙事文档的三个底层逻辑直接映射到论文笔记：

| Amazon 原则 | 论文 One-Pager 映射 |
|---|---|
| 每个词必须 earn its place | 一页上限强迫取舍——只留真正重要的 |
| 假设读者零上下文 | 用自己的话写，不抄 abstract |
| 先定义问题再讨论方案 | 先写 Why（C2），再写 How（C3） |

## 写作哲学

### 1. 写作强迫思考清晰

Amazon 用叙事文档替代 PPT 的核心原因：**写不清楚就是没想清楚。**
应用到论文笔记：如果你不能用自己的话在 C1 把论文讲清楚，说明你没真懂。

### 2. 文档是决策资产，不是汇报材料

Amazon 文档留存后成为新人入职时理解历史的入口。同样，你的论文
One-Pager 应该成为你两年后还能翻阅并快速重建理解的知识资产。

### 3. Truth-seeking，而非 prove-yourself-right

Amazon 文档的目的是寻找真相（这个项目该不该做），而非证明自己是对的。
应用到论文笔记：C6（我的判断）不应只夸论文好——必须同时指出局限、标注不信的数字。

## Weasel Words 禁令

以下词汇在 One-Pager 中禁止出现（来自 Amazon 写作文化）：

- "可能" / "或许" / "似乎" / "大概"
- "很多人认为" / "众所周知"
- "值得进一步研究"（不说具体研究什么 = 没想清楚）
- "in a sense" / "to some extent"

**替代法**：要么写"我认为 X，依据是 Y"，要么删掉。

## 强编辑原则

写完第一稿后：
1. 每一句话追问"删掉这句，读者还理解吗？"
2. 如果会 → 删掉
3. 如果不会 → 这句可以留，但检查能否更短

## 冷读测试

Amazon 要求文档在 meeting 开头 20 分钟默读——不提前发、不宣讲。
这意味着文档必须自成一体，不需要作者解释。

同理，你的 One-Pager 也必须接受冷读测试：
给一个完全不了解这篇论文的人看，看他能否准确说出论文做了什么、能信几分。

## PR/FAQ 思维

Amazon 的 PR/FAQ（Press Release + FAQ）是"逆向工作法"的核心工具：
先想象结果（客户拿到产品时的感受），再推导要做什么。

应用到论文阅读：
- **PR 部分 = C1（一句话）**：如果这篇论文是你写的，你会怎么宣布它的核心发现？
- **FAQ 部分 = C5（边界）+ C6（判断）**：如果你是 reviewer，最可能攻击哪些点？提前回答。

## 累积效应

Amazon 文档积累成组织的决策历史。10 篇 One-Pager 是笔记，30 篇是知识库，
100 篇是你对这个领域的**判断坐标系**。

达到一定数量后你会自然获得三种能力：
1. 看到新论文能立刻说出它和前人的真实区别（而非它声称的区别）
2. 能凭记忆串出一条领域发展脉络
3. Reviewer 指出一个 gap 时，你脑子里已经有 3 篇相关论文的 One-Pager
